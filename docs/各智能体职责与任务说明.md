# 优效营（uxyy）各智能体职责与任务说明

> **用途**：人手或 AI 智能体在开始开发前阅读，明确「谁负责哪里、交付什么、依赖谁」。  
> **权威流程与命令**：仍以根目录 **[多智能体并行开发指南.md](../多智能体并行开发指南.md)** 为准；**产品与表归属细节**以 **[优效营（uxyy.cn）小微企业一体化经营系统（MVP版）产品需求文档.md](../优效营（uxyy.cn）小微企业一体化经营系统（MVP版）产品需求文档.md)**（以下简称 PRD）为准。

---

## 一、所有智能体共同要做的事（Phase 0 + 日常）

| 事项 | 说明 |
|------|------|
| 对齐环境与命令 | Node 20、`pnpm`、`docker`（可选本地 PG/Redis），见指南 **§1.1、§2.4、§7.3** |
| Phase 0 参与 | 共同完成 **数据库 Schema 定版**（Drizzle + Migration）、**接口契约**（`docs/api` + Swagger DTO + `@uxyy/shared` Zod 对齐）、见指南 **第二章** |
| 分支与工作流 | 从 `develop` 拉 **`feature/<域>-<简述>`**，见指南 **§3.2**；合并前满足 **§6.2 / §6.4 门禁** |
| 契约与 Migration | HTTP 契约三件套同步；库表变更只用 **migration 分支/独立 PR**，见指南 **§5、§6.4** |
| 编码与 CI | 根目录 `pnpm run lint | typecheck | test`；PR 勾选模板中的检查项，见指南 **§3.3、§8.3** |

## 二、分层与依赖（谁可以依赖谁）

- **底座**：Agent-Auth（JWT、企业、RBAC、部分 OA）。
- **业务域**：CRM、Inventory、Finance **不互相直接深耦合**实现；需要通过 **已定版契约**、共享表约束或下层服务接口协作（参见 PRD 多智能体架构说明：同级避免直接反向依赖）。
- **前端**：编排 UI，依赖各域已暴露的 REST 契约；可辅以 Mock（指南 **§7.2**）。

**简略依赖链**：

```text
Agent-Auth →（无业务域前置）

Agent-CRM   → 依赖 Auth（租户/用户上下文）
Agent-Inventory → 依赖 Auth；业务上常与 CRM 单据关联，但通过 API/数据契约协作

Agent-Finance → 依赖 Auth；并与进销存单据统计等按 PRD 协同

Agent-AI → 依赖 Auth；与 Finance/Inventory 等按「异步任务/OCR 结果入账」契约协作

Agent-Frontend → 依赖上述各域接口进度（可先 Mock）
```

## 三、各智能体分工一览

### Agent-Auth（用户认证 · 权限 · 部分 OA）

| 维度 | 内容 |
|------|------|
| **主要职责** | 注册/登录、**Passport JWT**（`JwtStrategy`、`AuthGuard`、`@nestjs/jwt` 签发 Access/Refresh）、RBAC；企业切换；OA 中与认证/权限/审批框架强相关的能力（参见 PRD 对 `approval_*`、`roles`、`users` 等归属） |
| **后端目录** | `uxyy-api/src/modules/auth/`（及与 OA 共用审批模型时需在模块内收口，勿污染他域私有表逻辑） |
| **共享数据** | `users`、`enterprises`、`user_enterprises`、`roles`、`approval_flows`、`approval_records` 等 **PRD 11.5.2 表归属**：共享表由他方只读引用，**表结构变更需架构师把关** |
| **建议分支前缀** | `feature/auth-*` |
| **典型交付** | 认证 API、`JwtModule`/Passport 配置、Swagger 与安全相关 DTO、`@uxyy/shared` 中与 auth 契约相关的 Zod |
| **依赖** | 无（第一层）；需最先稳定「用户上下文」「enterprise_id」约定供全仓使用 |

---

### Agent-CRM（客户管理）

| 维度 | 内容 |
|------|------|
| **主要职责** | 客户档案、分类与标签、商机/跟进记录、与销售相关的客户侧能力（以 PRD CRM 章节为准） |
| **后端目录** | `uxyy-api/src/modules/crm/` |
| **私有表示例** | `customers`（PRD：CRM 全权） |
| **建议分支前缀** | `feature/crm-*` |
| **典型交付** | CRM 域 REST、`customers` 相关 Drizzle schema + Migration（独立 PR）、DTO/Swagger、共享 Zod |
| **依赖** | Agent-Auth（鉴权、`enterprise_id`、用户身份） |

---

### Agent-Inventory（进销存核心）

| 维度 | 内容 |
|------|------|
| **主要职责** | 商品/分类、采购订单、销售订单、库存流水与预警等（PRD 进销存章节） |
| **后端目录** | `uxyy-api/src/modules/inventory/` |
| **私有表示例** | `products`、`sales_orders`、`purchase_orders`、`inventory`、`inventory_logs` 等 |
| **建议分支前缀** | `feature/inventory-*` |
| **典型交付** | 进销存 API、/schema/migration（独立 PR）、与财务对接字段需在契约中与 Finance 对齐 |
| **依赖** | Agent-Auth；与 CRM **通过订单/客户外键契约协作**，勿直接依赖对方模块内部文件（指南 **§6.4**） |

---

### Agent-Finance（财务）

| 维度 | 内容 |
|------|------|
| **主要职责** | 发票、凭证、应收应付、报表等（PRD 财务章节） |
| **后端目录** | `uxyy-api/src/modules/finance/` |
| **私有表示例** | `invoices`、`voucher_entries`、`account_subjects` 等 |
| **建议分支前缀** | `feature/finance-*` |
| **典型交付** | 财务 API、migration、金额与期间类接口的 Swagger/Zod **需特别谨慎** |
| **依赖** | Agent-Auth；与 **Inventory** 在「单据→凭证」链路上按已定版契约配合 |

---

### Agent-AI（AI 智能层）

| 维度 | 内容 |
|------|------|
| **主要职责** | OCR/发票结构化、调用通义千问 / DeepSeek 等、智能记账建议、异步推理（宜用 **Redis + BullMQ**，见指南 **§10.3、§10.4**） |
| **后端目录** | `uxyy-api/src/modules/ai/`（或可拆 worker 包，仍以 Monorepo 约定为准） |
| **建议分支前缀** | `feature/ai-*` |
| **典型交付** | AI 编排服务、队列任务定义与幂等、`REDIS`/外部 API Key 不落库文档说明 |
| **依赖** | Agent-Auth；与 **Finance/Inventory** 通过「识别结果入账」等契约对接 |

---

### Agent-Frontend（前端 UI）

| 维度 | 内容 |
|------|------|
| **主要职责** | `uxyy-web`：页面、组件、TanStack Query 数据请求、表单与移动端适配（PRD 功能与路线图） |
| **目录** | `uxyy-web/`；合理使用 **`@uxyy/shared`**（Zod/常量/类型），避免复制后端 DTO |
| **建议分支前缀** | `feature/frontend-*` |
| **典型交付** | 页面与路由、`msw`/Mock（指南 §7.2）、与 Swagger 对齐的客户端类型（若团队启用生成流程） |
| **依赖** | 各后端域 **API 契约**；开发期可用 Mock **不阻塞**后端（指南 §7.2） |

## 四、与本项目工程栈的对应关系（摘自指南）

| 栈层 | Agent 侧常见工作 |
|------|------------------|
| Monorepo / pnpm | 在根目录跑统一脚本；为子包补齐 `lint` / `build` / `test` |
| Nest 模块边界 | 只在本 Agent 所属 `modules/<域>/` 内堆业务导出；跨域走约定接口 |
| 契约 | 改路由即对齐 **YAML（若保留） + Swagger + Zod**（指南 §2.2、§6.4） |
| Drizzle | 本域表迁移走 **`migration/*` PR**，见 §5 |
| Passport JWT | 主要由 **Agent-Auth** 维护；其余 Agent 路由使用 Guards，不复制鉴权内核 |

---

## 五、表所有权速查（与 PRD 11.5.2 一致）

遇到「谁能改这张表」时，以 PRD **11.5.2 表所有权**为准；以下为常见共享表摘要：

| 表/域 | Owner |
|------|-------|
| `users`、`enterprises`、`user_enterprises`、`roles` | **Agent-Auth**（共享，其他域勿改结构） |
| `customers` | Agent-CRM |
| `products`、订单与库存相关 | Agent-Inventory |
| `invoices`、`voucher_entries`、`account_subjects` | Agent-Finance |
| 审批骨架 `approval_*`（若按 PRD 归 Auth） | **Agent-Auth** 与各业务域通过流程配置协作 |
