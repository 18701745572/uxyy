## 📖 教程概述

本教程基于 **优效营 MVP 实际功能**，通过「小明便利店」真实业务案例，手把手教你掌握系统全部功能。

### 🎯 学习目标

1. 完成系统注册与登录，掌握用户资料查看、企业切换、成员邀请

3. 学会进销存：供应商、商品、采购/销售订单、仓库管理、库存盘点、库存预警
4. 理解财务：发票 OCR 识别、发票管理（核验/入账/作废）、凭证录入、应收应付、财务报表
5. 使用 OA：审批流程配置、请假、报销、考勤打卡、补卡申请与审批、员工通讯录
6. 体验 AI 助手：AI 任务、客户流失预警、商机成单预测

## 第一章：系统入门

### 1.1 注册与登录

#### 步骤 1：访问系统

```
本地开发地址：http://localhost:3001
生产环境：https://uxyy.cn
```

#### 步骤 2：注册账号

1. 点击「注册」或访问 `/register`
2. 填写表单：
   - **手机号**：`13800138001`（如已被占用，换其他号码）
   - **验证码**：开发环境输入任意 6 位数字（如 `123456`）
   - **密码**：`Dev12345!`（至少 8 位，含大小写+数字+特殊字符）
   - **企业名称**：`小明便利店`
3. 点击「注册」

> **💡 开发环境快捷登录**：若已运行 `pnpm run db:seed`，可直接使用种子账号：
> - 手机号：`13800138000`
> - 密码：`Dev12345!`

#### 步骤 3：登录系统

1. 访问 `/login`
2. 输入手机号和密码
3. 点击「登录」

**登录成功后会看到**：工作台首页，侧边栏包含：工作台、消息中心、客户管理、进销存管、财务管理、OA 办公、AI 智能

### 1.2 工作台首页

**场景**：快速了解企业经营状况，处理待办事项

**操作步骤**：

1. 登录后自动进入工作台 (`/dashboard`)
2. 查看 **经营概览** 区域：
   - **今日销售额**：当日实时销售金额
   - **待处理订单数**：待审核/待出库的订单数量
   - **库存预警数量**：低于安全库存的商品数量

3. 查看 **快捷入口** 区域（根据权限显示）：
   - 「新建客户」→ 快速创建 CRM 客户
   - 「新建订单」→ 快速创建销售订单
   - 「录入发票」→ 快速录入财务发票

4. 查看 **待办事项** 区域（根据权限显示）：
   - **待审批**：需要审批的请假、报销、补卡申请
   - **待跟进客户**：超过跟进周期的客户
   - **库存预警**：库存不足的商品列表

5. **快捷操作**：
   - 点击待办事项可直接跳转到处理页面
   - 点击经营数据可查看详情趋势

### 1.3 消息中心

**场景**：查看系统通知、审批提醒、预警信息

**操作步骤**：

1. 点击侧边栏「消息中心」或顶部导航栏的铃铛图标
2. 进入消息中心页面 (`/dashboard/notifications`)
3. 查看消息分类：
   - **全部消息**：所有类型的通知
   - **未读消息**：未查看的新消息
   - **系统通知**：系统公告、更新提醒
   - **审批提醒**：待审批事项提醒
   - **预警通知**：库存预警、价格预警等

4. **消息操作**：
   - 点击消息标题查看详情
   - 点击「标记已读」清除未读状态
   - 点击「全部已读」一键清除所有未读
   - 点击「删除」移除单条消息

5. **实时推送**：
   - 系统支持 WebSocket 实时推送
   - 新消息到达时，铃铛图标显示红点提醒
   - 点击铃铛可快速预览最新消息

**常见消息类型**：
| 消息类型 | 触发场景 | 处理方式 |
|---------|---------|---------|
| 审批提醒 | 有员工提交请假/报销申请 | 点击跳转审批页面 |
| 库存预警 | 商品库存低于安全线 | 点击跳转采购建议 |
| 价格预警 | 采购均价波动超过阈值 | 点击查看价格分析 |
| 经营洞察 | 每周自动生成经营报告 | 点击查看详细分析 |
| 系统公告 | 系统更新、维护通知 | 阅读后标记已读 |

### 1.4 用户资料与企业切换

登录后进入 **工作台** (`/dashboard`)，页面展示：

| 区域 | 内容 |
|------|------|
| 经营概览 | 今日销售额、待处理订单数、库存预警数量 |
| 快捷入口 | 新建客户、新建订单、录入发票 |
| 待办事项 | 待审批、待跟进客户、库存预警 |

### 1.4 用户资料与企业切换

**场景**：查看用户 ID（用于商机负责人、客户归属销售），或切换企业

**操作步骤**：

1. 点击侧边栏底部 **「用户资料」**，进入 `/dashboard/profile`
2. 在 **账号信息** 区域查看：
   - **用户 ID**：数字 ID（如 `1`），用于 CRM 中的负责人/归属销售字段
   - **当前企业**：显示当前登录的企业名称和 ID
3. 在 **企业列表** 区域：
   - 若有多家企业，点击「切换」按钮切换当前企业
   - 切换后所有数据（客户、订单等）会刷新为新企业数据

### 1.5 数据备份与恢复

**场景**：定期备份企业数据，防止数据丢失

**权限要求**：老板（boss）或行政（oa）角色

**操作步骤**：

1. 点击侧边栏底部 **「用户资料」**
2. 在 **数据备份** 区域，点击「备份管理」
3. 进入备份管理页面 (`/dashboard/profile/backup`)

**创建备份**：
1. 点击「立即备份」按钮
2. 选择备份类型：
   - **全量备份**：备份所有企业数据
   - **企业备份**：仅备份当前企业数据
3. 等待备份完成（显示进度条）
4. 备份成功后，文件会显示在备份列表中

**查看备份列表**：
| 字段 | 说明 |
|-----|------|
| 备份时间 | 备份创建时间 |
| 备份类型 | 全量/企业 |
| 文件大小 | 压缩后的文件大小 |
| 状态 | 已完成/进行中/失败 |
| 操作 | 下载/恢复/删除 |

**恢复数据**：
1. 在备份列表中找到需要恢复的备份
2. 点击「恢复」按钮
3. **⚠️ 警告**：恢复会覆盖现有数据，系统会要求确认
4. 输入确认码（如 `RESTORE`）
5. 等待恢复完成

**自动备份设置**：
1. 点击「自动备份设置」
2. 配置参数：
   - **启用自动备份**：开启/关闭
   - **备份频率**：每天/每周/每月
   - **备份时间**：选择执行时间（建议凌晨 2-4 点）
   - **保留天数**：自动删除超过保留期的备份（建议 30 天）
3. 点击「保存设置」

**导出 JSON 数据**：
1. 点击「导出数据」按钮
2. 选择要导出的数据类型：
   - 客户数据
   - 商品数据
   - 订单数据
   - 发票数据
   - 全部数据
3. 系统生成 JSON 格式文件，可下载到本地

**导入 JSON 数据**：
1. 点击「导入数据」按钮
2. 上传之前导出的 JSON 文件
3. 系统会显示预览，确认数据无误后点击「确认导入」
4. **注意**：导入会合并数据，如遇 ID 冲突会自动跳过

### 1.6 企业成员管理

**权限要求**：老板（boss）或行政（oa）角色

**操作步骤**：

1. 在 **用户资料** 页，点击 **「企业与成员」→「企业成员管理」**
2. 进入 `/dashboard/profile/enterprise-members`
3. **添加成员**（已注册用户）：
   - 点击「添加成员」
   - 输入已注册同事的手机号
   - 选择角色：财务 / 销售 / 仓管 / 行政
   - 点击「确认」
4. **生成邀请链接**（未注册用户）：
   - 点击「生成邀请链接」
   - 选择预设角色
   - 复制链接发送给同事
   - 同事访问 `/join?t=xxx` 可完成注册并自动加入企业

> **权限说明**：
> - 老板：拥有所有权限
> - 财务：`finance:*` + `crm:read`
> - 销售：`crm:*` + `inventory:read`
> - 仓管：`inventory:*`
> - 行政：`oa:*` + `system:member`

## 第二章：数据导入导出

### 2.1 批量导入数据

**场景**：从 Excel 批量导入客户、商品、订单等数据

**支持导入的数据类型**：
- 客户数据
- 供应商数据
- 商品数据
- 销售订单
- 采购订单
- 库存盘点数据
- 发票数据
- 财务凭证
- 会员等级
- 客户分类
- 员工档案

**操作步骤（以导入客户为例）**：

1. 点击侧边栏「客户管理」→「客户列表」
2. 点击页面右上角「导入」按钮
3. 在弹窗中：
   - **步骤 1**：下载模板文件
     - 点击「下载模板」获取标准 Excel 模板
     - 查看模板中的字段说明和示例数据
   - **步骤 2**：填写数据
     - 按照模板格式填写客户信息
     - 确保必填字段完整（客户名称、手机号等）
     - 保存为 .xlsx 或 .csv 格式
   - **步骤 3**：上传文件
     - 点击「选择文件」或拖拽文件到上传区域
     - 支持 .xlsx、.xls、.csv 格式
     - 文件大小不超过 10MB
4. 点击「开始导入」

**导入过程**：
1. 系统显示导入进度条
2. 自动验证数据格式
3. 显示导入结果：
   - 成功导入：XX 条
   - 导入失败：XX 条
   - 重复跳过：XX 条
4. 点击「查看详情」可下载错误报告

**导入规则**：
| 规则 | 说明 |
|-----|------|
| 重复检测 | 根据手机号/编码自动检测重复 |
| 数据验证 | 必填字段、格式、范围校验 |
| 批量大小 | 单次最多导入 5000 条 |
| 错误处理 | 部分失败不影响其他数据 |

### 2.2 批量导出数据

**场景**：导出数据用于分析、备份或对接其他系统

**操作步骤（以导出客户为例）**：

1. 点击侧边栏「客户管理」→「客户列表」
2. 可选：使用筛选条件缩小导出范围
   - 按客户分类筛选
   - 按归属销售筛选
   - 按创建时间筛选
3. 点击页面右上角「导出」按钮
4. 选择导出格式：
   - **Excel (.xlsx)**：推荐，保留格式和公式
   - **CSV (.csv)**：通用格式，兼容性好
   - **JSON (.json)**：用于系统对接
5. 选择导出字段：
   - 「导出全部字段」或
   - 「自定义字段」勾选需要的字段
6. 点击「确认导出」
7. 等待生成文件，自动下载到本地

**导出选项**：
| 选项 | 说明 |
|-----|------|
| 导出范围 | 全部数据 / 当前页 / 选中行 |
| 导出格式 | Excel / CSV / JSON / PDF |
| 字段选择 | 全部字段 / 自定义字段 |
| 编码格式 | UTF-8（默认）/ GBK |

**各模块导出功能**：
| 模块 | 支持导出的数据 |
|-----|---------------|
| 进销存 | 产品、供应商、销售订单、采购订单、库存盘点 |
| 财务 | 发票、凭证、银行流水 |
| CRM | 客户、会员等级、客户分类 |
| OA | 员工档案、考勤记录 |

### 2.3 导入导出最佳实践

**导入前准备**：
1. **备份数据**：导入前建议先创建备份
2. **小批量测试**：首次导入先用 10-20 条测试
3. **数据清洗**：在 Excel 中先清理异常数据
4. **字段映射**：确保字段名称与模板一致

**常见问题处理**：
| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 手机号格式错误 | 包含空格或特殊字符 | 使用 Excel 的「查找替换」清理 |
| 日期格式错误 | 非标准日期格式 | 统一设置为 YYYY-MM-DD 格式 |
| 编码错误 | 中文字符乱码 | 保存为 UTF-8 编码 |
| 必填字段为空 | 数据不完整 | 补充必填字段后重新导入 |

## 第二章：客户管理（CRM）

3. 点击右上角「新建」按钮
4. 填写客户信息：

| 字段 | 填写值 | 说明 |
|------|--------|------|
| 客户名称 | `王老板` | 必填 |

| 电话 | `13900139000` | 可选 |
| 地址 | 浙江省杭州市余杭区 | 可选 |

   - **类型**：自定义（可选：自定义/成交状态/行业/区域）
   - **描述**：`月采饮料类稳定、议价空间大`
   - **颜色**：`#1890ff`（蓝色）

**后续操作**：在客户编辑页面，可将客户关联到此分类

### 2.3 创建商机

3. 点击「新增商机」

| 商机名称 | `批量采购饮料` |
| 描述 | 100箱饮料，下月提货 |
| 状态 | 潜在（可选：潜在/意向/报价/成交/售后/流失） |
| 预计金额 | `5000` |
| 预计成交日期 | `2026-06-15` |
| 成交概率 | `60`（%） |

## 第六章：系统安全与设置

### 6.1 安全功能说明

优效营系统内置多层安全防护机制，保护您的企业数据安全。

#### 6.1.1 登录安全

**密码要求**：
- 至少 8 位字符
- 包含大小写字母
- 包含数字
- 包含特殊字符（如 !@#$%）

**登录频率限制**：
- 连续 5 次登录失败，账户锁定 15 分钟
- 系统会记录登录 IP 和时间

**会话管理**：
- Access Token 有效期：2 小时
- Refresh Token 有效期：7 天
- 长时间未操作自动退出

#### 6.1.2 CSRF 防护

系统自动防护跨站请求伪造攻击：
- 所有非安全请求（POST/PUT/DELETE）自动验证 CSRF Token
- Token 存储在 Cookie 中，每次请求自动携带
- 无需手动操作，系统自动处理

#### 6.1.3 请求频率限制

防止暴力破解和恶意请求：
| 接口类型 | 限制频率 | 说明 |
|---------|---------|------|
| 登录接口 | 5次/分钟 | 防止密码暴力破解 |
| 注册接口 | 3次/分钟 | 防止批量注册 |
| 敏感操作 | 10次/分钟 | 密码修改、转账等 |
| 普通接口 | 100次/分钟 | 一般数据操作 |

**触发限流后**：
- 返回 429 错误码
- 响应头显示剩余等待时间
- 等待时间结束后自动恢复

### 6.2 系统监控

**场景**：查看系统运行状态和性能指标

**权限要求**：老板（boss）角色

**操作步骤**：

1. 点击侧边栏「用户资料」
2. 在 **系统监控** 区域，点击「查看指标」
3. 进入监控页面，查看以下信息：

**HTTP 请求统计**：
- 总请求数
- 成功请求数（2xx）
- 客户端错误（4xx）
- 服务端错误（5xx）
- 平均响应时间

**数据库性能**：
- 查询总数
- 慢查询数（>1000ms）
- 平均查询时间

**业务指标**：
- 用户登录次数
- 数据导出次数
- 备份任务次数

**慢请求告警**：
- 响应时间超过 1 秒的请求会被记录
- 在日志中标记为 WARN 级别
- 可用于性能优化分析

### 6.3 日志查看

**场景**：排查问题、审计操作记录

**权限要求**：老板（boss）或行政（oa）角色

**操作步骤**：

1. 点击侧边栏「用户资料」
2. 在 **系统日志** 区域，点击「查看日志」
3. 进入日志查看页面

**日志筛选**：
- **时间范围**：今天/最近7天/最近30天/自定义
- **日志级别**：DEBUG/INFO/WARN/ERROR
- **操作类型**：登录/数据操作/系统操作
- **用户**：按用户筛选

**日志字段**：
| 字段 | 说明 |
|-----|------|
| 时间 | 操作发生时间 |
| 级别 | DEBUG/INFO/WARN/ERROR |
| 用户 | 操作用户 ID 和姓名 |
| IP 地址 | 客户端 IP |
| 操作 | 具体操作描述 |
| 请求 ID | 唯一请求标识 |
| 耗时 | 操作耗时（毫秒） |

**敏感信息保护**：
- 密码字段自动脱敏显示为 `***`
- Token 字段自动脱敏
- 银行账号只显示后 4 位

### 6.4 安全建议

**密码安全**：
1. 定期更换密码（建议 90 天）
2. 不同系统使用不同密码
3. 避免使用生日、手机号等简单密码
4. 启用双因素认证（如支持）

**数据安全**：
1. 定期备份数据（建议每日自动备份）
2. 重要操作前创建备份点
3. 离职员工及时移除系统权限
4. 敏感数据导出后妥善保管

**网络安全**：
1. 使用 HTTPS 访问系统
2. 不在公共 WiFi 下登录系统
3. 及时退出不使用的会话
4. 发现异常登录及时修改密码

## 第八章：常见问题与故障排查

### 8.1 登录问题

#### 问题：忘记密码
**解决方案**：
1. 在登录页面点击「忘记密码」
2. 输入注册手机号
3. 获取短信验证码
4. 设置新密码（需符合密码强度要求）
5. 使用新密码登录

#### 问题：账户被锁定
**原因**：连续 5 次登录失败
**解决方案**：
1. 等待 15 分钟后自动解锁
2. 或联系企业管理员重置密码

#### 问题：提示 "请求过于频繁"
**原因**：触发频率限制
**解决方案**：
1. 等待 1 分钟后重试
2. 检查是否有自动化脚本在登录

### 8.2 数据问题

#### 问题：导入数据失败
**排查步骤**：
1. 检查文件格式是否为 .xlsx 或 .csv
2. 检查必填字段是否完整
3. 检查数据格式是否正确（日期、数字等）
4. 下载错误报告查看具体原因

#### 问题：数据不显示
**排查步骤**：
1. 检查是否切换了正确的企业
2. 检查筛选条件是否设置正确
3. 刷新页面或重新登录
4. 检查用户权限是否足够

#### 问题：导出数据为空
**排查步骤**：
1. 检查筛选条件是否过于严格
2. 检查是否有数据权限
3. 尝试导出全部数据（不设置筛选）

### 8.3 性能问题

#### 问题：页面加载缓慢
**解决方案**：
1. 检查网络连接
2. 清除浏览器缓存
3. 使用 Chrome 或 Edge 浏览器
4. 避免在高峰期操作大量数据

#### 问题：操作响应慢
**可能原因**：
- 数据量过大（单次查询超过 10000 条）
- 复杂报表计算
- 网络延迟

**优化建议**：
1. 使用筛选条件缩小数据范围
2. 分页查看数据
3. 导出后在本地分析大数据量

### 8.4 备份恢复问题

#### 问题：备份失败
**排查步骤**：
1. 检查磁盘空间是否充足
2. 检查数据库连接是否正常
3. 查看系统日志中的错误信息
4. 尝试手动执行 pg_dump 测试

#### 问题：恢复后数据不完整
**可能原因**：
- 备份文件损坏
- 备份时数据正在写入
- 恢复过程中断

**解决方案**：
1. 验证备份文件完整性（校验和）
2. 使用更早的备份点恢复
3. 联系技术支持

### 8.5 联系技术支持

**问题反馈渠道**：
1. **系统内反馈**：点击用户资料 → 帮助与反馈
2. **邮件支持**：support@uxyy.cn
3. **电话支持**：400-XXX-XXXX（工作日 9:00-18:00）

**反馈时请提供**：
- 问题描述和复现步骤
- 错误截图
- 浏览器版本
- 企业 ID 和用户 ID（可在用户资料中查看）
