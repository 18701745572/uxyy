# 优效营（uxyy.cn）系统操作教程

> 实例驱动学习 - 从零开始掌握系统全部功能

---

## 📖 教程概述

本教程通过一个**虚拟企业案例**，带您逐步体验优效营系统的核心功能。

### 🎯 学习目标

1. 完成系统注册与登录，并能在「用户资料」中查看 **用户 ID**、切换企业
2. 掌握 CRM：客户列表、分类、商机、跟进与 AI 话术
3. 学会进销存：供应商、商品、采购/销售订单、仓库、**盘点**、**库存预警**
4. 理解财务：**发票全流程**、凭证列表、应收应付、**财务报表（多报表页签）**
5. 使用 OA：**审批流程配置**、请假、报销、考勤、**补卡申请与补卡审批**、审批待办、「员工通讯录」浏览
6. 体验 **AI 助手**：**AI 任务**、流失预警、商机成单预测等页签（`/dashboard/ai`，以侧边栏文案为准）
7. （拓展）任选 **「深度实例：端到端业务剧本」**（§D.1～D.5）中的一条，完成跨模块连贯操作

### 🏢 案例背景

**企业名称**：小明便利店  
**行业**：零售/批发  
**规模**：3家分店，5名员工  
**业务场景**：日常销售、采购补货、会员管理、财务记账

## 第一章：系统入门

### 1.1 注册与登录

**步骤1：访问系统**
```
地址：http://localhost:3000
```

**步骤2：注册账号**
1. 点击「注册」或打开注册页 `/register`
2. 填写**手机号**（示例：`13800138000`，请勿与已注册账号冲突）
3. **验证码**：当前前端提示为「开发环境输入任意值」；请按注册页说明通过校验即可（具体规则以 `register` 表单校验为准）
4. 设置**密码**（须满足「至少 8 位」等规则，示例：`Dev12345!`）
5. **企业名称**：`小明便利店`
6. 点击「注册」

**步骤3：登录系统**
1. 打开 `/login`，输入手机号和密码
2. 点击「登录」

> **本地开发种子账号（可选）**：若您运行过后端 `seed`，默认示例用户一般为手机 `13800138000`、密码 `Dev12345!`（亦可通过环境变量 `SEED_DEV_PHONE` / `SEED_DEV_PASSWORD` 覆盖）。与教程自拟注册不冲突，任选其一即可登录体验。

### 1.2 首页概览

登录后请进入 **工作台 / Dashboard**，侧边栏通常包含：首页、客户管理、进销存、财务、OA 办公、**AI 助手** 等（**以您当前版本界面为准**，名称可能是「AI 助手」或与产品文案一致）。

**模拟看板（教学示意，非真实运行结果）**：若首页有经营概览类卡片，可自行对照下列数字理解「小店一天长什么样」——

| 指标（示例） | 模拟值 |
|--------------|--------|
| 今日销售额 | ¥8,260 |
| 待处理采购单 | 2 张 |
| 待跟进客户 | 5 人（含「王老板」） |
| 库存预警 SKU | 1 个（如矿泉水低于下限） |

首页卡片与看板指标可能随迭代变化；**若界面与上表不一致，以您当前环境为准**。

### 1.3 用户资料与企业切换

**场景**：需要查看 CRM「负责人 / 归属销售」里要填写的 **数字用户 ID**，或在一家账号下关联了多家企业时切换到另一家企业。

**操作步骤**：
1. 登录后点击侧栏底部的 **「用户资料」**，进入 `/dashboard/profile`（任意业务页也可从同一入口进入）。
2. 在 **账号信息** 中查看：
   - **用户 ID**：与 JWT 载荷中的 `sub` 一致，供商机负责人、客户归属销售等表单填写；
   - **当前企业 ID**：当前上下文绑定企业。
3. 在 **企业列表** 中：若名下有多家企业，可对非当前行点击 **「切换」**，成功后会刷新数据上下文（工作台与客户/进销存等列表均以新企业为准）。

**备注**：本节路径与字段以当前前端为准；若没有多企业条目，一般为注册时仅此一家 enterprise。

### 1.4 企业成员管理（与暂未上线能力）

若您以 **老板（boss）或行政（oa）** 登录，在 **用户资料** 页可使用 **「企业与成员」→ 企业成员管理**，将 **已在优效营注册**的同事手机号加入当前企业，并设为 **财务 / 销售 / 仓管 / 行政** 四类角色之一。*该能力仅维护 `user_enterprises` 关系，不等同于 OA「员工通讯录」建档（后者需成员已存在）。*

**当前版本明确不做**（亦无产品入口或 API）：**① 企业主转让**（更换 `enterprises.owner_id` 的完整治理流程）；**②** 其他未在 PRD §9.26 与本发版说明列出的成员治理能力。  
**已实现**：给指定手机号生成**统一邀请**（`enterprise_invitations`）：未注册用户打开 `/join?t=…` 可走受限注册并入会；已注册用户登录后可**接受邀请**并入会。详情请见 `docs/api-documentation.md` 与企业成员管理页「生成邀请链接」。

## 第二章：客户管理

### 2.1 创建客户

**场景**：小明便利店新来了一位大客户「王老板」，需要录入系统

**操作步骤**：
1. 点击左侧菜单「客户管理」
2. 在客户管理首页，点击「客户列表」卡片
3. 进入客户列表页面后，点击右上角「新建」按钮
4. 填写信息（完整字段）：
   - **客户名称**（必填）：`王老板`
   - **联系人**（可选）：`张三`
   - **电话**（可选）：`13900139000`
   - **地址**（可选）：`浙江省杭州市余杭区`
   - **客户类型**（可选）：`企业客户`（企业客户/个人客户）
   - **客户等级**（可选）：`VIP`（VIP/普通/潜在）
   - **行业**（可选）：`零售`
   - **标签**（可选）：`重点客户`, `货到付款`
   - **来源**（可选）：`手动录入`（手动录入/导入/微信/其他）
   - **归属销售 ID**（可选）：填写负责跟进的销售员在用户体系中的**数字用户 ID**（与「设置 / 用户资料」中显示的「用户 ID」一致）；不填表示暂不指定归属销售
   - **信用额度**（可选）：`50000`
   - **备注**（可选）：`重点客户，货到付款`
5. 点击「创建」

**效果**：客户档案创建成功，可在列表中查看与继续维护。

> **💡 字段说明**：
> - `客户类型`：区分企业客户和个人客户，影响后续业务处理
> - `客户等级`：VIP客户可享受会员价和专属服务
> - `标签`：用于快速分类和筛选客户
> - `信用额度`：设置客户的赊账上限，超限时系统自动预警
> - **归属销售 ID**：对应数据库中的内部用户主键（`users.id`），不是手机号；给自己归类时可填当前账号在「用户资料」里看到的用户 ID
>
> **💡 关于会员积分**：当前 MVP 中，**新建客户不会自动创建「会员档案」记录**；会员等级与积分在「会员管理」等能力中单独维护。教程里说的「从 0 分起步」指业务含义上的新客，若需可查可兑的积分，请在会员相关菜单中为客户建档或积分入账。

### 2.2 客户分类

填写下列步骤时，可与文首 **「模拟数据速查」** 表中 CRM 区「客户分类」一行对照。

**场景**：小明便利店希望把客户按业务维度打上统一标签（例如「批发大客户」），便于列表筛选与后续运营。

**操作步骤**：
1. 点击左侧菜单「客户管理」
2. 在客户管理首页，点击「**客户分类**」卡片（说明文字为「按状态、行业、区域等维度分类客户」）
3. 进入页面 `客户分类` 后，点击右上角「**新增分类**」按钮（带加号图标）
4. 在弹窗「**新增客户分类**」中填写（与当前前端表单一致）：
   - **分类名称**（必填）：如 `批发大客户`
   - **类型**（可选）：默认「**自定义**」；亦可选用「成交状态」「行业」「区域」（与后端枚举、列表中的类型展示一致）
   - **描述**（可选）：如 `月采饮料类稳定、议价空间大`
   - **颜色**（可选）：取色器或十六进制，默认 `#1890ff`
   - **排序**（可选）：整数；**新建时界面默认即为 `0`**（与截图一致）。若需区分多条记录在列表中的先后顺序，可改为其他数字；未改则保持 `0` 保存即可
5. 点击「**保存**」

**效果**：新分类出现在表格中（列：**分类名称 / 类型 / 描述 / 颜色 / 排序 / 操作**）。可在行末使用 **编辑**（铅笔图标）、**删除**（垃圾桶图标；删除前会弹出确认）。

> **说明**：本页维护的是企业维度的「客户分类」主数据；若需在客户档案中关联具体分类，请以客户新建/编辑表单中的相关字段为准（以当前版本界面为准）。

### 2.3 创建商机

**场景**：王老板表示下个月要批量采购100箱饮料

**操作步骤**：
1. 点击左侧菜单「客户管理」
2. 在客户管理首页，点击「商机管理」卡片
3. 点击右上角「新增商机」按钮，在弹窗中填写（与当前前端表单一致）：
   - **关联客户**（必填）：须从下拉框选择已有客户（例如刚创建的 `王老板`）；未选择客户无法保存
   - **商机名称**（必填）：`批量采购饮料`
   - **描述**（可选）：如 `100箱饮料，下月提货`
   - **状态**（可选）：默认「潜在」；可选「潜在 / 意向 / 报价 / 成交 / 售后 / 流失」
   - **预计金额**、**实际金额**（可选）：如预计金额填 `5000`
   - **预计成交日期**、**实际成交日期**（可选）：日期选择器，例如预计成交 `2026-06-15`
   - **成交概率 (%)**（可选）：`0`～`100`，默认 `0`，可按判断自行填写
   - **负责人（用户 ID，可选）**：与「归属销售 ID」相同，填企业成员的**用户 ID**（见「用户资料」）；不填表示暂不指定负责人
   - **备注**（可选）
4. 点击「保存」

**效果**：商机创建成功并出现在列表中，可查看状态、预计金额、概率、预计成交日、负责人等。

#### 3.2.1 新建采购单（草稿）

**路径**：进销存 → **采购订单**（约 `/dashboard/inventory/purchase-orders`）→ **+ 新建采购订单**。

**操作步骤**：
1. 点击左侧菜单「进销存」
2. 点击「**采购订单**」卡片进入列表
3. 点击「**+ 新建采购订单**」
4. 填写表单（数值可照 **文首「模拟数据速查」** 中「采购单行」一行抄写）：
   - **供应商**（必选）：下拉选择 `旺旺食品`（提交到后端的字段为 **`supplierId`**，对应选项里的 `#id`，无需手填）
   - **采购明细**：「添加商品行」后，选择商品 `可乐`，数量 `50`，单价 `45`（单价允许为 `0`；行数据结构为 **`productId`、`quantity`、`unitPrice`**）
   - **备注**（可选）：如 `首批铺货`（对应后端 **`remark`**）
5. 点击「**创建**」

**结果**：后端会生成单号（如 `PO` 开头）、**状态为「草稿」**（`draft`）。此时**尚未增加库存**，也**未进入审批队列**。

> **编辑草稿**：列表中 **草稿** 行可点「**编辑**」。当前 Web 与后端约定为：**编辑时一般只更新备注**；供应商与明细行已在建单时确定，若需大改，更稳妥的做法是 **删除该草稿后重新建单**（以您当前界面提示为准）。

#### 3.2.2 状态一览（与列表筛选项一致）

| 界面显示 | 后端状态值 | 含义（业务上） |
|----------|------------|----------------|
| 草稿 | `draft` | 可编辑备注、可删除、可「提交」送审 |
| 待审批 | `pending` | 已提交，等待「批准」或「拒绝」 |
| 已批准 | `approved` | 审批通过；**满足条件时可执行入库**（部分入库后单据可仍为已批准，直至收齐） |
| 已完成 | `completed` | 各明细行已全部收齐（`received` 达到订购数量），采购闭环结束 |
| 已取消 | `cancelled` | 审批拒绝或业务作废后的终态之一 |

列表顶部的 **「全部状态」** 下拉框与上表一一对应，便于按状态筛选。

#### 3.2.3 列表上的按钮分别做什么（当前 Web）

以下描述与 **采购订单列表行内按钮** 一致（若您本地界面有迭代，以实际为准）：

| 按钮 | 何时出现 | 作用（与后端行为对应） |
|------|----------|-------------------------|
| **编辑** | 仅 **草稿** | 修改备注等（见 §3.2.1 说明） |
| **提交** | 仅 **草稿** | 将状态从 **`draft` → `pending`（待审批）**，表示「正式送入审批流程」；**不是入库、也不会自动加库存** |
| **删除** | 仅 **草稿** | 物理删除该单及明细（后端限制：非草稿不可删） |
| **批准** | **待审批** | **`pending` → `approved`**，采购单可被入库 |
| **拒绝** | **待审批** | **`pending` → `cancelled`**（视为审批不通过） |
| **取消** | **待审批** 或 **已批准** | 作废单据（后端不允许对 **已完成 / 已取消** 再次作废） |

**小结**：**「提交」= 从草稿正式进入「待审批」**；批准后才会走到可执行入库的 **「已批准」** 状态。

#### 3.2.4 入库与库存（批准后）

**业务含义**：货到仓之后，需要将实收数量记进系统：**只有状态为「已批准」 (`approved`) 的采购单，后端才允许执行入库**。支持按采购明细 **分行、分多次入库**（部分到货）；当**每一行**的已累计实收数量 **等于** 该行订购数量时，整单变为 **「已完成」** (`completed`)。全部收齐后，系统**可能触发采购相关自动记账**（记账失败通常**不阻断**入库结果，具体以服务端配置与日志为准）。

**接口**（便于联调或对账）：`PUT /api/v1/inventory/purchase-orders/:id/inbound`，请求体为 **`{ "items": [ { "itemId": <采购明细 id>, "inboundQty": <本次入库数量> }, ... ] }`**（camelCase，与后端 DTO 一致）。

> **说明（Web 界面）**：仓库中已为上述入库能力提供前端 API 封装，但 **采购订单列表页未必展示「入库」按钮**。若您在浏览器里走完「提交 → 批准」后**看不到入库入口**，需要依赖 **接口文档（如 `docs/api-documentation.md`）**、自建操作入口或后续版本的界面更新；**库存是否增加，以是否在已批准状态下成功执行入库为准**。
>
> 「扫码入库」等若为教学场景或非本仓库已实现能力，请以实际上线菜单为准。

#### 3.2.5 本节练习小结

按顺序完成：**新建供应商与商品 → 新建采购订单（草稿）→ 提交（待审批）→ 批准（已批准）→（如环境支持）入库至完成**。若仅练习到「已批准」，也已完成主流程理解与列表操作。

**效果**：在成功完成 **已全部收齐的入库** 后，对应 SKU 库存增加；价税层面的财务凭证是否自动生成，参见 §3.2.4「自动记账」说明及财务模块文档。

**路径**：进销存 → **库存盘点**（`/dashboard/inventory/stocktaking`）。

**操作步骤**：

## 第四章：财务管理

### 4.1 发票录入（新建、OCR 与手工）

**场景**：收到供应商或客户侧发票，需在系统登记备查，并走 **未核验 → 已核验 → 已入账**（或作废）闭环。

**路径**：财务 → **发票管理**（`/dashboard/finance/invoices`）→ **「+ 新建发票」**。

**操作步骤**：
1. 打开发票管理页，点击 **「+ 新建发票」**，进入 **「新建发票」** 表单。
2. （可选）上传票面图并 **「识别票面」**：将 OCR 结果合并进表单（清晰度与格式限制以页面提示为准，失败时可改 **手工录入**）。
3. **手工核对**下列常见字段（可与文首模拟表「不含税 / 税 / 价税合计」对照，金额以实际票面为准）：
   - 发票号码、代码、类型、**开票日期**（日期控件，一般选 `YYYY-MM-DD`）
   - 不含税金额、税率、税额、价税合计；购/销方名称与税号等
   - **关联来源**（可选）：如与进销订单关联，按表单说明选择 `sourceType` / `sourceId`（以当前界面字段为准）
4. 点击表单底部 **「创建」**。

> **重要（与接口一致）**：提交到后端的 **开票日期字段名为 `issueDate`**（由页面将您选择的日期转成 ISO  UTC 当日午夜等格式）。表单中的 **「备注」仅作本地填写，当前版本不会随创建/保存请求发往服务端**——若需票面备注落库，须待产品增加对应字段与迁移。

**可选：独立 OCR 练习页**  
财务首页 → **「发票OCR识别」**卡片（路径一般为 `/dashboard/finance/invoices/ocr`）——与发票管理内嵌识别二选一即可，以您环境卡片链接为准。

### 4.2 发票核验、入账与作废

发票保存后默认 **未核验**。在 **发票管理列表** 每一行（以状态为准）可使用：

| 按钮 / 能力 | 典型前置状态 | 说明 |
|-------------|-------------|------|
| **编辑** | 未核验 | 修改票面字段（与创建时相同业务含义） |
| **核验** | 未核验 → 已核验 | 业务上确认票面信息无误 |
| **入账** | 已核验 | 进入 **已入账**，并可能触发 **自动生成凭证**（以服务端逻辑为准） |
| **作废** | 未核验 / 已核验（见界面是否开放） | **已入账** 单据一般不可再编辑，作废规则以列表按钮与接口为准 |
| **删除** | 仅未核验 | 物理删除该条记录（谨慎操作） |

**练习建议**：创建一张 §4.1 模拟金额的发票 → **核验** → 到 **§4.3 凭证录入** 中按来源筛选 **`invoice`（发票入账）** 观察是否出现新凭证。

### 5.3 查看审批状态（列表筛选）

**场景**：申请人或督办人想知道 **请假、报销、补卡** 是否已被通过。

**当前 Web（MVP）常用方式**：
1. **请假**：「OA 办公」→ **「请假管理」** `/dashboard/oa/leave-requests`：顶部筛选 **「待审批 / 已通过 / 已驳回」**，点击 **「查看详情」** 进入 `/dashboard/oa/leave-requests/{id}` 阅读字段与状态。
2. **报销**：**「报销管理」** `/dashboard/oa/expense-requests` —— 同理筛选状态后进入详情。
3. **补卡**：
   - **申请人**：以考勤页补卡申请与列表展示为准（若有「我的补卡」或状态列，可核对是否仍为待审批）；
   - **审批人**：使用 **§5.1.2 补卡审批** 页，对待办直接使用 **「通过 / 驳回」** 按钮完成审批（**无需**再走单独请假式详情页，除非产品后续合并「统一待办」）。

> **说明**：请假/报销 **详情页** 在当前版本中 **多为只读展示**；若在列表或详情 **未看到「同意 / 驳回」按钮**，审批动作可在 **OA 工作台**（具备 **`oa:approve`** 时顶部的「待您审批」入口）、**「待审批中心」**（`/dashboard/oa/pending-approvals`）或 **补卡审批** 页完成（见 **§5.1.2**）。请勿将 OA 首页 **「本人的申请进度」** 数字误认为「待人审批队列」——后者仅对有审批权限账号单独展示。

### 5.4 报销申请

**场景**：员工小王报销差旅费

**操作步骤**：
1. 点击左侧菜单「OA办公」
2. 在OA办公首页，点击「报销管理」卡片
3. 进入报销管理页面后，点击右上角「新建报销」按钮
4. 填写信息：
   - 类型：`差旅费`
   - 金额：`500`
   - 上传凭证图片（若表单支持 OCR / 附件，以页面为准）
5. 点击「提交」

### 5.5 员工通讯录

**场景**：按部门或关键字浏览本企业 **员工档案**（来源于 OA 员工扩展与账号资料）。

5. 若无管理权限，页面为只读；开发环境可运行 `pnpm db:seed` 自动为种子用户写入一条示例档案；亦可使用接口 `POST /api/v1/oa/employee-profiles` 建档。

## 第六章：AI智能功能

### 6.1 页面结构：三个主 Tab

侧边栏打开 **AI 助手** → 进入 **`/dashboard/ai`**。页面上方有三块卡片式 Tab：

| Tab | 说明 |
|-----|------|
| **AI 任务** | 提交异步任务（如会计分录建议、分类等）；可查看队列状态与输出（详见页内表单与列表） |
| **客户流失预警** | 基于客户与订单特征的流失风险视图（详见页内图表/列表） |
| **商机成单预测** | 对商机做概率或风险档位展示（与服务端模型或规则一致） |

同一时间仅展开一个 Tab 的内容；**请以您环境中三个 Tab 的真实标签文案为准**（早期文档曾把部分能力散落在 CRM 小节，本章按当前聚合页收口）。

### 6.2 AI 任务与凭证联动

**场景**：希望由 AI 建议分录写入账套。

**操作步骤**：
1. 在 **`/dashboard/ai`** 选择 **「AI 任务」** Tab。
2. 按页面说明选择任务类型、填写 payload（若有客户/发票/摘要等字段）。
3. 提交任务后轮询或通过列表查看 **completed** 状态及 **输出**；成功时可能在 **§4.3 凭证录入** 中出现来源为 **「AI 任务」** 的记录（取决于任务类型与后端实现）。
4. 亦可从 **凭证录入页**顶部链接 **「从 AI 助手写入凭证」** 跳回本页。

### 6.3 客户流失预警（AI 模块内）

**操作步骤**：
1. 切换到 **「客户流失预警」** Tab。
2. 阅读页内 KPI、列表或空态说明；若有 **「刷新 / 运行检测」** 类按钮，按提示执行。
3. 表中风险等级与建议文案 **以服务端返回为准**；可与 **§2.6** CRM 侧的同类叙述对照理解——**数据源可能一致，入口不同**。

### 6.4 商机成单预测（AI 模块内）

**操作步骤**：
1. 切换到 **「商机成单预测」** Tab。
2. 若有商机下拉或批量预测入口，选中 **§2.3** 中创建的「批量采购饮料」等记录，触发预测。
3. 对照 **CRM 商机手工填写的「成交概率 (%)」**（§2.3）与本页 AI 输出，思考差异来源（特征、口径、更新时间）。

### 6.5 CRM 侧的 AI 话术（交叉引用）

商机详情页的 **AI 话术推荐** 仍为日常销售最常用的生成式辅助，步骤见 **§2.4**；**与本章 Tab 互不替代**——话术偏「怎么说」，商机预测 Tab 偏「胜率与策略数字」。

### 6.6 库存与采购分析（仍以进销存与报表为准）

**说明**：独立的「一键智能采购建议」列表若未在侧边栏单独出现，可将 **§3.5 库存预警**、**财务报表经营仪表盘中的低库存与热销列表**（§4.5）视作补货决策参考；下表仍为 **教学示意**。

## 深度实例：端到端业务剧本

下列 **5 个剧本** 在前文分章功能之上，用**同一条故事线**把多模块串起来；每一步括号内为对应**细读章节**，便于跳回查字段与按钮。

### D.1 剧本一：新客「刘老板」从建档到首单意向

**角色**：店长（销售兼维护）  
**时间线**：约 40 分钟（含等待与翻页）

| 顺序 | 业务动作 | 操作提示（实例数据见文首速查表） |
|------|----------|----------------------------------|
| 1 | 记下自己的 **用户 ID** | §1.3，用于客户「归属销售」 |
| 2 | **新建客户**「刘老板」，填电话、等级、归属销售 ID | §2.1 |
| 3 | 新建 **客户分类** 或选用已有「批发大客户」 | §2.2 |
| 4 | 建 **商机**「夏季饮料增量」，预计金额、预计成交日 | §2.3 |
| 5 | 进商机 **详情**，用 **AI 话术** 场景「谈判协商」生成沟通素材 | §2.4 |
| 6 | 写一条 **跟进记录**，约定下次沟通时间 | §2.5 |

**收尾**：在商机列表把状态改为「报价」或「意向」，便于 §D.4 从报表侧看销售漏斗（若报表含商机口径，以实际为准）。

### D.2 剧本二：库存告急 → 采购 → 核验发票

**角色**：库管 + 财务（可同一人扮演）  
**前提**：§3.1 已建 SKU、§3.0 已建供应商。

| 顺序 | 业务动作 | 章节 |
|------|----------|------|
| 1 | 「库存预警」点 **立即检查**，确认某 SKU 待处理 | §3.5 |
| 2 | **采购订单**：草稿 → **提交** → **批准**（勿与入库混淆） | §3.2 |
| 3 | 若环境支持，按 §3.2.4 完成 **入库**；否则理解接口含义即可 | §3.2.4 |
| 4 | **新建发票**（采购侧），**核验 → 入账** | §4.1～§4.2 |
| 5 | 「凭证录入」按来源 **`invoice`** 核对；「应收应付」看是否从列表核销 | §4.3～§4.4 |

### D.3 剧本三：忘打卡 → 补卡申请 → 审批通过

**角色**：员工 **A**（申请）、审批人 **B**（老板或 oa；须 **`oa:approve`**）  
**若仅有一个测试账号**：可先提交补卡，再在同一「老板」账号下完成审批（真实环境多为两人）。

| 顺序 | 业务动作 | 章节 |
|------|----------|------|
| 1 | A：考勤页 **补卡申请** 标签，填日期、上/下班、原因 | §5.1.1 |
| 2 | B：「补卡审批」页筛选 **待审批** | §5.1.2 |
| 3 | B：对行点击 **通过**（或 **驳回** 演练拒绝路径） | §5.1.2 |
| 4 | A：回到考勤/个人记录，确认与当日记录一致（以系统回写为准） | §5.1 |

### D.4 剧本四：财务月末 45 分钟核对清单

**角色**：财务  
**可将本表打印为自检表**：

1. **发票管理**：未核验是否清零；已入账是否与纸票一致（§4.1～§4.2）  
2. **凭证录入**：按来源筛 **销售单 / 采购单 / 发票入账**，抽查借贷金额（§4.3）  
3. **应收应付**：是否仅剩合理在途；已开票未回款是否与客户解释一致（§4.4）  
4. **财务报表**：同一 **会计月份** 下，经营仪表盘与利润表 **收入/成本** 口头可解释（§4.5）  
5. **AI 智能纠错**：批量检测后记录问题条数，测试环境再点「一键修复」（§4.6）  
6. **税务数据准备**：导出思路见 §4.7，申报仍在税局网站完成  

### D.5 剧本五：AI 助手「一单到底」

**角色**：店长 / 财务

| 顺序 | 业务动作 | 章节 |
|------|----------|------|
| 1 | `/dashboard/ai` → **商机成单预测**，对照 §2.3 手工概率 | §6.4 |
| 2 | **客户流失预警** 跑检测，与 §2.6 叙事对照 | §6.3 |
| 3 | **AI 任务** 生成与凭证相关输出，回 **凭证录入** 查来源 **AI 任务** | §6.2、§4.3 |

> **说明**：D.1～D.5 均为 **练习增强**；任一步若与本地 MVP 菜单不一致，以侧栏与 **「教程与当前版本」** 表格为准。

## 📝 练习任务

> **提示**：下列步骤中的名称、编码、金额可与文首 **「📋 模拟数据与实例速查」** 表对照填写；若提示「已存在」，可在同一表规律下略改编码或名称。

### 任务1：完整销售流程
1. （可选）先到 **§1.3** 记下当前账号 **用户 ID**，回填到客户「归属销售」（§2.1）。
2. 点击「客户管理」→「客户列表」→「新建」，创建客户「刘老板」（电话等可按速查表中王老板同款或自拟）
3. 确保进销存中已有可售商品（建议按 **§3.1 示例 A** 或速查表建「550ml 矿泉水」SKU：`SKU-WATER-550`、单价 `36` 等）
4. 点击「进销存」→「**销售订单**」→「新建」（或「+ 新建销售单」，**以按钮文案为准**），向「刘老板」销售 20 箱矿泉水，单价可按 `36` 或与商品档案一致
5. 建议先按 **§3.4.1** 建仓库，再尝试 **§3.4.3** 跑一次盘点草稿，或使用 **§3.5「立即检查」** 生成库存预警并处理状态。
6. 点击「财务」→「**凭证录入**」按来源 **`sales_order`** 等筛选查看是否有关联凭证（§4.3）

### 任务2：采购补货流程
1. 点击「进销存」→「**供应商管理**」→「+ 新建供应商」，创建「农夫山泉」（字段同 **§3.0**；电话、地址可照速查表中旺旺食品格式自拟）
2. 点击「进销存」→「**商品管理**」→「+ 新建商品」，为后续采购准备 SKU（字段同 **§3.1**；例如编码 `SKU-WATER-550`、名称 `550ml 矿泉水`、销售单价 `36`、单位 `箱`）
3. 点击「进销存」→「**采购订单**」→「+ 新建采购订单」，选择该供应商并添加商品行（如矿泉水 50 箱），点击「创建」得到 **草稿**；再按列表上的 **提交 → 批准** 推进状态（详见 **§3.2**，尤其 **§3.2.3「提交」≠ 入库**）
4. **入库与库存**：若界面无「入库」入口，参见 **§3.2.4**（API 或通过其他环境完成）；完成后可用 **§3.4.3 盘点** 或库存相关页核对数量
5. 「财务」→「发票管理」→ **+ 新建发票**，录入采购侧发票并完成 **核验 → 入账**（§4.2）；再到「应收应付」「凭证录入」交叉验证（§4.4、§4.3）

### 任务3：客户分类（与 §2.2 对齐）
1. 「客户管理」→「客户分类」
2. 点击「新增分类」，新增一条示例分类（如名称「教程-批发客户」、类型「自定义」）
3. 在列表中确认出现该行；可尝试编辑后保存，或删除测试数据

### 任务4：商机与 AI 话术（与 §2.3、§2.4 对齐）
1. 「客户管理」→「商机管理」，新建或选择一条商机
2. 点击商机名称进入详情页 `/dashboard/crm/opportunities/{id}`（步骤同 §2.3、§2.4）
3. 打开「AI 话术推荐」，场景选「谈判协商」，生成并阅读话术

### 任务5：库存盘点闭环（§3.4.3）
1. 确认已完成 **§3.4.1** 仓库、**§3.1** 至少一个 SKU，并完成一次 **入库或销售出库**（使系统有存量可查）。
2. 「进销存」→「库存盘点」→ **+ 新建盘点单**，选对仓库与范围。
3. 打开草稿单 → 行末 **录入** → 修改 **实盘数量** → **保存** → **确认盘点**。

### 任务6：财务 KPI 一日游（§4.3～§4.5）
1. 录入或选用已有 **已核验发票**（§4.1～4.2），打开「应收应付」看余额（§4.4）。
2. 打开「财务报表」，依次切换五个页签并记录同一月份下仪表盘与三张表的一致性（§4.5）。
3. 打开「AI 智能纠错」对 **任一凭证 ID** 做 **单次检测**（§4.6）。

### 任务7：OA 配置 + AI 三联（§5.0 & §6.x）
1. 按 §5.0 新建 **请假** 类审批流并 **启用**。
2. 「AI 助手」中依次点开 **流失预警**、**商机预测** Tab，跑一次空载/实测并记录与服务端的差异提示。

### 任务8：补卡申请与审批（§5.1.1～§5.1.2）
1. 使用 **员工视角** 登录（或同一测试账号）→「OA」→「考勤管理」→ **「补卡申请」** 标签，提交一条上班卡补卡（日期、原因可照文首速查表）。
2. 使用具备 **OA 审批权限** 的账号（通常为 **老板**；或已授予 `oa:approve` 的 **行政** 账号）→ OA 首页 **「补卡审批」** 或考勤页 **「补卡审批」** 按钮，进入 `/dashboard/oa/attendance/make-up-approvals`。
3. 筛选 **待审批**，对该条点击 **通过**；再回到申请人视角确认记录与状态（以实际界面为准）。
4. （扩展）将筛选改为 **已驳回**，自拟一条申请后由审批人点 **驳回**，走完整拒绝路径。

### 任务9：深度剧本任选（附录 §D.1～§D.5）
从 **「深度实例：端到端业务剧本」** 中任选 **D.1**（新客意向）或 **D.2**（采购+发票）或 **D.4**（月末核对）之一，**不打断**地按表格顺序做完，并在文中记录遇到的与本地版本不一致之处。
