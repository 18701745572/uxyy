# 优效营 · 用户操作手册（详细版）

**面向对象**：企业管理人员、财务人员、销售人员、仓管人员、行政人员等日常使用系统的全体员工  
**使用环境**：Windows / macOS / Linux + Chrome / Edge / Safari 等现代浏览器  
**阅读时间**：约 30 分钟（可收藏后按需查阅）

## 一、系统概述

### 1.1 什么是优效营？

**优效营（uxyy.cn）** 是一套专为小微企业设计的 **一体化经营管理系统**。它将企业日常经营中的多个业务环节整合到统一平台，实现数据互通、流程协同、智能辅助。

**核心价值**：
- **数据统一**：客户、订单、库存、财务数据互联互通，避免信息孤岛
- **流程规范**：标准化业务流程，减少人为差错
- **智能辅助**：AI 技术辅助识别、预测、决策，提升工作效率
- **实时洞察**：经营数据实时汇总，辅助管理决策

### 1.2 系统能解决什么问题？

| 业务痛点 | 优效营解决方案 |
|---------|---------------|
| 客户信息分散在 Excel、微信、笔记本中 | 统一客户档案，跟进记录全程留痕 |
| 销售、库存、财务数据对不上 | 业务单据自动生成财务凭证，数据一致 |
| 月底对账耗时耗力 | 应收应付自动统计，对账单一键生成 |
| 发票录入容易出错 | OCR 自动识别，准确率 95%+ |
| 库存经常断货或积压 | 库存预警自动提醒，智能补货建议 |
| 员工请假报销流程混乱 | 线上审批流，进度实时可查 |
| 经营情况不清楚 | 多维度报表，可视化数据看板 |

## 二、快速入门

### 2.1 首次使用准备

**硬件要求**：
- 电脑：Windows 7+ / macOS 10.12+ / Linux
- 内存：4GB 及以上
- 屏幕分辨率：1280×768 及以上（推荐 1920×1080）

**软件要求**：
- 浏览器：Chrome 90+ / Edge 90+ / Safari 14+ / Firefox 88+
- 网络：稳定的互联网连接（推荐带宽 10Mbps 以上）
- 插件：建议启用 JavaScript 和 Cookie

**账号准备**：
- 向系统管理员申请账号
- 确认已分配企业权限
- 记录系统登录网址（建议收藏到浏览器书签）

### 2.2 登录系统

**步骤详解**：

1. **打开浏览器**
   - 推荐使用 Chrome 或 Edge 浏览器
   - 在地址栏输入系统网址（如 `https://uxyy.cn`）

2. **进入登录页面**
   - 页面显示登录表单
   - 包含：手机号输入框、密码输入框、登录按钮

3. **输入登录信息**
   - 手机号：输入管理员为您开通的手机号
   - 密码：输入初始密码（首次登录后建议立即修改）

4. **点击登录**
   - 系统验证账号密码
   - 验证通过后进入工作台

**首次登录后必做**：
1. 修改初始密码（系统可能强制要求）
2. 完善个人资料（姓名、部门等）
3. 查看当前所属企业是否正确
4. 熟悉界面布局

### 2.3 注册新账号（如开放）

若贵单位开放自助注册：

1. 在登录页面点击「注册账号」
2. 填写注册信息：
   - 手机号（必填，用于登录）
   - 验证码（点击获取，输入短信验证码）
   - 密码（需符合安全要求）
   - 确认密码
   - 姓名（必填）
3. 阅读并同意用户协议
4. 点击「注册」
5. 注册成功后，**联系管理员分配企业权限**

> **注意**：注册后若看不到业务数据，是因为尚未分配企业权限，请联系单位负责人。

## 三、工作台使用指南

### 3.1 工作台概述

工作台是登录后的默认页面，提供 **一站式经营概览**，帮助您快速了解企业运营状况。

### 3.2 工作台组成

#### 3.2.1 经营概览卡片

显示关键经营指标：

| 指标 | 说明 | 数据来源 |
|-----|------|---------|
| 今日销售额 | 当日已确认的销售订单金额 | 销售订单 |
| 今日采购额 | 当日已确认的采购订单金额 | 采购订单 |
| 待处理订单 | 待审核/待出库的订单数量 | 销售订单+采购订单 |
| 库存预警 | 低于安全库存的商品数量 | 库存管理 |
| 应收账款 | 客户欠款总额 | 销售订单-回款 |
| 应付账款 | 欠供应商总额 | 采购订单-付款 |

#### 3.2.2 快捷入口

常用功能的快速链接：
- 新建客户
- 新建订单
- 录入发票
- 查看报表
- 发起审批

#### 3.2.3 待办事项

需要您处理的事项列表：
- **待审批**：需要您审批的请假、报销、补卡申请
- **待跟进**：超过跟进周期的客户
- **库存预警**：库存不足的商品
- **应收提醒**：即将逾期或已逾期的应收账款

#### 3.2.4 最近动态

显示最近的操作记录：
- 最新订单
- 最新客户
- 最新审批

### 3.3 工作台操作示例

**场景：上班后查看今日经营情况**

1. 登录系统，自动进入工作台
2. 查看「今日销售额」了解当日业绩
3. 查看「待处理订单」了解需要处理的业务
4. 点击「待审批」卡片查看待审批事项
5. 点击「库存预警」查看需要补货的商品
6. 根据需要点击快捷入口开始工作

## 四、消息中心使用指南

### 4.1 消息中心概述

消息中心集中管理所有系统通知，支持实时推送，确保重要信息不遗漏。

### 4.2 消息类型

| 消息类型 | 触发场景 | 处理方式 |
|---------|---------|---------|
| 审批提醒 | 有员工提交请假/报销/补卡申请 | 点击跳转审批页面 |
| 审批结果 | 您的申请被通过/驳回 | 点击查看详情 |
| 库存预警 | 商品库存低于安全线 | 点击跳转采购建议 |
| 价格预警 | 采购均价波动超过阈值 | 点击查看价格分析 |
| 经营洞察 | 每周自动生成经营报告 | 点击查看详细分析 |
| 系统公告 | 系统更新、维护通知 | 阅读后标记已读 |
| 任务完成 | AI 任务处理完成 | 点击查看结果 |

### 4.3 消息中心操作

**查看消息**：
1. 点击侧边栏「消息中心」或顶部铃铛图标
2. 消息列表按时间倒序排列
3. 未读消息显示红点标记

**消息筛选**：
- 全部消息
- 未读消息
- 系统通知
- 审批提醒
- 预警通知

**消息操作**：
- 点击消息标题查看详情
- 点击「标记已读」清除未读状态
- 点击「全部已读」一键清除
- 点击「删除」移除消息

### 4.4 实时推送

系统支持 WebSocket 实时推送：
- 新消息到达时，铃铛图标显示红点
- 浏览器通知（需授权）
- 点击铃铛可快速预览最新消息

## 五、客户管理（CRM）详细指南

### 5.1 客户管理概述

客户管理模块帮助企业建立完整的客户档案，跟踪销售机会，维护客户关系，提升成交转化率。

### 5.2 客户列表

#### 5.2.1 功能说明

集中管理所有客户信息，支持搜索、筛选、排序。

#### 5.2.2 客户字段

| 字段 | 说明 | 必填 |
|-----|------|------|
| 客户名称 | 客户公司或姓名 | 是 |
| 手机号 | 主要联系电话 | 是 |
| 客户分类 | 如：大客户、普通客户、潜在客户 | 否 |
| 归属销售 | 负责该客户的销售人员 | 否 |
| 客户等级 | A/B/C/D 级 | 否 |
| 客户来源 | 如：网络、转介绍、展会 | 否 |
| 地址 | 客户地址 | 否 |
| 备注 | 其他信息 | 否 |

#### 5.2.3 操作步骤

**新建客户**：
1. 点击「客户管理」→「客户列表」
2. 点击「新建客户」按钮
3. 填写客户信息
4. 点击「保存」

**搜索客户**：
1. 在搜索框输入客户名称或手机号
2. 支持模糊搜索
3. 按回车或点击搜索按钮

**筛选客户**：
- 按客户分类筛选
- 按归属销售筛选
- 按客户等级筛选
- 按创建时间筛选

**编辑客户**：
1. 在列表中点击客户名称
2. 进入客户详情页
3. 点击「编辑」按钮
4. 修改信息后保存

**删除客户**：
1. 进入客户详情页
2. 点击「删除」按钮
3. 确认删除（有订单关联的客户通常不能删除）

### 5.3 商机管理

#### 5.3.1 功能说明

管理销售机会，跟踪商机进展，预测成交金额。

#### 5.3.2 商机阶段

| 阶段 | 说明 | 成交概率 |
|-----|------|---------|
| 潜在 | 初步接触，了解需求 | 10% |
| 意向 | 客户有明确意向 | 30% |
| 报价 | 已提供报价方案 | 50% |
| 谈判 | 正在商务谈判 | 70% |
| 赢单 | 成功签约 | 100% |
| 输单 | 未能成交 | 0% |

#### 5.3.3 操作步骤

**新建商机**：
1. 点击「客户管理」→「商机管理」
2. 点击「新建商机」
3. 选择关联客户
4. 填写：
   - 商机标题
   - 预计金额
   - 当前阶段
   - 预计成交日期
   - 商机描述
5. 点击「保存」

**更新商机阶段**：
1. 进入商机详情页
2. 点击阶段进度条
3. 选择新阶段
4. 系统自动记录阶段变更历史

**查看商机详情**：
- 基本信息
- 阶段历史
- 关联跟进记录
- AI 成单预测

### 5.4 跟进记录

#### 5.4.1 功能说明

记录与客户的每次沟通，形成完整的客户互动历史。

#### 5.4.2 跟进类型

| 类型 | 说明 |
|-----|------|
| 电话沟通 | 电话交流 |
| 上门拜访 | 面对面拜访 |
| 微信/QQ | 在线沟通 |
| 邮件 | 邮件往来 |
| 其他 | 其他方式 |

#### 5.4.3 操作步骤

**新建跟进记录**：
1. 在客户详情页点击「新建跟进」
2. 或点击「客户管理」→「跟进记录」→「新建跟进」
3. 选择客户
4. 填写：
   - 跟进时间（默认当前时间）
   - 跟进类型
   - 跟进内容
   - 下次跟进提醒（可选）
5. 点击「保存」

**使用 AI 话术推荐**：
1. 在商机详情页点击「AI 话术推荐」
2. 选择场景：
   - 首次接触
   - 跟进回访
   - 报价沟通
   - 谈判协商
   - 促成成交
3. 系统生成推荐话术
4. 可复制使用或参考修改

### 5.5 客户分类

#### 5.5.1 功能说明

对客户进行分组管理，便于针对性营销。

#### 5.5.2 操作步骤

**创建分类**：
1. 点击「客户管理」→「客户分类」
2. 点击「新建分类」
3. 输入分类名称和描述
4. 点击「保存」

**为客户打标签**：
1. 进入客户详情页
2. 点击「编辑」
3. 在「客户分类」字段选择分类
4. 保存

### 5.6 会员管理

#### 5.6.1 功能说明

管理客户会员资格，设置会员权益。

#### 5.6.2 会员等级

系统支持多级会员体系：
- 普通会员
- 银卡会员
- 金卡会员
- 钻石会员

#### 5.6.3 操作步骤

**配置会员等级**：
1. 点击「客户管理」→「会员等级」
2. 点击「新建等级」
3. 设置：
   - 等级名称
   - 等级图标
   - 升级条件（累计消费金额）
   - 折扣率
   - 积分倍数
   - 专属权益
4. 点击「保存」

**为客户开通会员**：
1. 进入客户详情页
2. 点击「开通会员」
3. 选择会员等级
4. 设置有效期
5. 点击「确认」

**查看会员积分**：
1. 在客户详情页查看积分余额
2. 查看积分明细（获得/消费记录）

### 5.7 AI 辅助功能

#### 5.7.1 客户流失预警

**功能说明**：
AI 自动分析客户行为数据，识别可能流失的客户。

**操作步骤**：
1. 点击「AI 智能」→「客户流失预警」
2. 点击「运行检测」
3. 查看高风险客户列表：
   - 客户名称

- 推荐新品/服务

#### 5.7.2 商机成单预测

### 5.8 最佳实践

1. **及时录入**：见完客户后 24 小时内录入跟进记录
2. **定期复盘**：每周查看商机漏斗，调整跟进策略
3. **善用 AI**：定期运行流失预警和成单预测
4. **分类管理**：根据客户价值进行分类，差异化服务
5. **数据完整**：尽量填写完整客户信息，便于后续分析

**功能说明**：
定期盘点库存，核对账实是否一致。

**盘点流程**：
1. **创建盘点单**
   - 选择盘点仓库
   - 选择盘点范围（全部/部分商品）
   - 系统生成盘点清单

2. **录入实盘数量**
   - 实际清点各商品数量
   - 在系统中录入实盘数

3. **生成盘盈盘亏**
   - 系统自动对比账面数和实盘数
   - 生成盘盈（实盘>账面）或盘亏（实盘<账面）记录

4. **确认盘点**
   - 审核盘点结果
   - 系统自动调整库存

**操作步骤**：
1. 点击「进销存管」→「库存盘点」
2. 点击「新建盘点单」
3. 选择盘点仓库和商品范围
4. 点击「生成盘点清单」
5. 录入实盘数量
6. 点击「生成盘盈盘亏」
7. 查看差异，确认无误后点击「确认盘点」

#### 6.6.5 库存预警

**功能说明**：
自动监控库存水平，及时提醒补货或处理积压。

**预警类型**：
| 类型 | 触发条件 | 处理建议 |
|-----|---------|---------|
| 库存不足 | 库存 < 库存下限 | 及时采购补货 |
| 库存积压 | 库存 > 库存上限 | 促销清仓 |
| 缺货预警 | 库存 = 0 | 紧急采购 |

**查看预警**：
1. 点击「进销存管」→「库存预警」
2. 查看预警商品列表
3. 点击商品可直接跳转采购建议

### 6.7 最佳实践

1. **及时入库**：商品到货后 24 小时内完成入库
2. **先进先出**：出库时优先发出先入库的商品
3. **定期盘点**：建议每月至少盘点一次
4. **设置预警**：为所有商品设置合理的库存上下限
5. **账实相符**：发现差异及时查明原因并调整

**支持发票类型**：
- 增值税普通发票
- 增值税专用发票
- 电子发票
- 通用机打发票

**识别字段**：
- 发票代码、发票号码
- 开票日期
- 购买方信息（名称、税号）
- 销售方信息（名称、税号）

   - 绿色：识别置信度高
   - 黄色：识别置信度中，建议核对
   - 红色：识别置信度低，必须核对
6. 点击「填入新建发票」自动跳转发票创建页
7. 补充或修正信息后保存

**识别准确率**：
- 印刷体发票：95%+

## 十二、故障排查与常见问题

### 12.1 登录问题

#### 问题：忘记密码
**解决方案**：
1. 在登录页面点击「忘记密码」
2. 输入注册手机号
3. 获取短信验证码
4. 设置新密码（需符合密码强度要求）
5. 使用新密码登录

#### 问题：账户被锁定
**现象**：提示「账户已锁定，请15分钟后重试」
**原因**：连续 5 次登录失败
**解决方案**：
1. 等待 15 分钟后自动解锁
2. 或联系企业管理员重置密码

#### 问题：提示 "请求过于频繁"
**现象**：返回 429 错误码
**原因**：触发频率限制
**解决方案**：
1. 等待 1 分钟后重试
2. 检查是否有自动化脚本在登录

#### 问题：登录后跳回登录页
**可能原因**：
- Cookie 被禁用
- 浏览器隐私模式
- Token 过期

**解决方案**：
1. 检查浏览器是否启用 Cookie
2. 关闭隐私/无痕模式
3. 清除浏览器缓存后重试
4. 更换浏览器尝试

### 12.2 页面显示问题

#### 问题：页面空白或加载不出来
**排查步骤**：
1. 检查网络连接
2. 按 F5 刷新页面
3. 按 Ctrl+F5 强制刷新（清除缓存）
4. 检查浏览器控制台是否有报错（F12）
5. 尝试更换浏览器

#### 问题：页面布局错乱
**可能原因**：
- 浏览器版本过低
- 缩放比例不正确
- 分辨率过低

**解决方案**：
1. 升级浏览器到最新版本
2. 按 Ctrl+0 恢复默认缩放
3. 调整屏幕分辨率至 1280×768 以上

#### 问题：按钮点击无反应
**排查步骤**：
1. 等待页面完全加载
2. 检查是否有错误提示
3. 查看浏览器控制台报错
4. 刷新页面后重试

### 12.3 数据问题

#### 问题：数据不显示或显示不全
**排查步骤**：
1. 检查是否切换了正确的企业
2. 检查筛选条件是否设置正确
3. 清除筛选条件查看全部数据
4. 检查用户权限是否足够
5. 刷新页面或重新登录

#### 问题：导入数据失败
**排查步骤**：
1. 检查文件格式是否为 .xlsx 或 .csv
2. 检查文件大小是否超过 10MB
3. 检查必填字段是否完整
4. 检查数据格式是否正确（日期、数字等）
5. 下载错误报告查看具体原因

**常见导入错误**：
| 错误提示 | 原因 | 解决方案 |
|---------|------|---------|
| 手机号格式错误 | 包含空格或特殊字符 | 使用 Excel 清理数据 |
| 日期格式错误 | 非标准日期格式 | 统一设置为 YYYY-MM-DD |
| 编码错误 | 中文字符乱码 | 保存为 UTF-8 编码 |
| 必填字段为空 | 数据不完整 | 补充必填字段 |

#### 问题：导出数据为空
**排查步骤**：
1. 检查筛选条件是否过于严格
2. 检查是否有数据权限
3. 尝试导出全部数据（不设置筛选）
4. 检查数据是否存在

### 12.4 功能使用问题

#### 问题：找不到某个功能
**排查步骤**：
1. 确认是否有该功能的权限
2. 检查是否在正确的菜单下
3. 使用搜索功能查找
4. 联系管理员确认功能是否开通

#### 问题：无法保存数据
**可能原因**：
- 必填字段未填写
- 数据格式不正确
- 网络连接中断
- 权限不足

**解决方案**：
1. 检查必填字段是否完整
2. 检查数据格式
3. 检查网络连接
4. 查看错误提示信息

#### 问题：审批流程卡住
**排查步骤**：
1. 查看当前审批节点
2. 联系当前审批人
3. 检查审批流程配置
4. 联系管理员处理

### 12.5 性能问题

#### 问题：页面加载缓慢
**解决方案**：
1. 检查网络连接速度
2. 清除浏览器缓存
3. 关闭不必要的浏览器标签页
4. 使用 Chrome 或 Edge 浏览器
5. 避免在高峰期操作大量数据

#### 问题：操作响应慢
**可能原因**：
- 数据量过大（单次查询超过 10000 条）
- 复杂报表计算
- 网络延迟

**优化建议**：
1. 使用筛选条件缩小数据范围
2. 分页查看数据
3. 导出后在本地分析大数据量
4. 避开业务高峰期

### 12.6 打印问题

#### 问题：打印格式错乱
**解决方案**：
1. 使用浏览器的打印预览功能
2. 调整打印设置（纸张大小、边距）
3. 导出 PDF 后打印
4. 使用系统提供的打印模板

### 12.7 移动端问题

#### 问题：手机访问页面显示不正常
**解决方案**：
1. 使用手机自带浏览器或 Chrome
2. 确保浏览器为最新版本
3. 开启横屏模式查看
4. 使用电脑端获得最佳体验

## 附：财务人员速查

### 从「建议」到「凭证库」

便于与 **产品需求文档** 中「**财务人员只需审核确认，凭证同步至凭证库**」一句话对齐，建议按下面顺序理解职责：

1. **系统或 AI**：给出识别结果、推荐科目或分录建议（含自动记账、AI 任务结果等）。  
2. **财务审核**：核对业务真实性、科目与金额是否正确；如需审批，在审批流程中 **同意 / 驳回** 并留痕。  
3. **凭证入库**：审核通过后，分录以正式凭证形式进入 **凭证库**，并与发票、订单等关联。

若某一步在本单位尚未开通，请向管理员确认路线图；**不要**在未核对的情况下将 AI 或规则引擎的输出直接作为最终账务依据对外报送。
